# v5 설계 스펙: 과거 매출 데이터 기반 단가 조회 시스템

작성일: 2026-04-16
상태: 확정 (brainstorming 완료)
Phase: 레벨 1 (조회 전용 데모)

---

## 한 줄 요약

직원이 거래처와 품명을 검색하면, 과거 5.5년치 매출 데이터에서 비슷한 거래를 찾아서 "언제, 어떤 규격으로, 얼마에 나갔는지" 보여주는 조회 화면.

---

## 배경

### 문제
직원이 견적을 낼 때마다 거래처별 단가를 몰라서 대표에게 물어봄.

### 왜 기존 접근이 실패했는가
- v1 (엑셀식 평탄화): UI 문제가 아니라 데이터 문제였음
- v2 (과거 견적 복사): 제작품이라 같은 조합이 거의 없어서 재사용 실패
- v3 (원가 계산기): 재료/가공 다양성이 넓어 분해 불가능
- v4 (결재 연동 점진 축적): 처음부터 쌓아야 해서 6개월간 대표 의존 지속
- 공통 원인: 전부 "사전 구축" 접근. 리바트 매핑 v1~v4의 반복 실패가 증거.

### v5가 다른 이유
5.5년치 매출 데이터가 이미 엑셀로 존재한다는 발견. 대표 머릿속 지식이 이미 "실제 거래 기록"으로 외부화되어 있음. 사전 세팅 노동 0. 검색만 가능하게 만들면 됨.

## 적재 규칙

### DB 위치
기존 `업무데이터.db`에 `sales_history` 테이블 추가

### 저장 항목 (행 하나당)
- 원본 그대로: 거래처, 날짜, 품명, 규격, 수량, 단가, 금액, 프로젝트, 납품처
- 자동 추출: 기본 품명 (+ 앞부분), 부가 토큰 목록 (+ 뒷부분들), 두께, 가로/세로/깊이

### 파싱 방식 (미니멀)
- 품명: 그대로 저장 + `+`로 분리해서 기본 품명 / 부가 토큰 저장
- 규격: `*`로 분리해서 가로/세로/깊이 저장
- 두께: 품명에서 `숫자t` 패턴 추출 (예: 3t → 3mm)
- 재질/가공/부자재 사전: 만들지 않음

### 제외 항목 (DB에 안 넣음)
- 전표합계, 운반비, 배송비, 택배, 퀵
- 외상매출금, 입금, 출금, 지급
- 설치비, 인건비 (매번 달라서 참고 가치 없음)
- 품명 없는 행, 단가 0인 행

## 검색 동작

### 검색 입력
- 거래처: 검색창 (일부만 쳐도 자동 매칭. 예: `라코` → `라코스`)
- 품명: 검색창 (자유 입력)

### 매칭 순서 (자동 단계별)
1. 정확히 같은 품명 + 같은 거래처
2. 비슷한 품명(접두어 매칭) + 같은 거래처 → 예: `3t포맥스` 검색 시 `3t포맥스+양면테이프`도 나옴
3. 같은 품명 + 다른 거래처 (참고용, 접힌 상태)

### 정렬
최근 거래가 위로

### 결과 표시 항목
날짜 / 품명 / 규격 / 수량 / 단가 / ㎡당 단가 (면적 품목만)

---

## ㎡당 단가 표시 규칙

### 계산
단가 ÷ (가로mm × 세로mm ÷ 1,000,000) = ㎡당 단가

### 각 줄마다 독립 계산
같은 기본 품명이라도 부가 항목(+양면테이프, +타공 등)이 다르면 ㎡당 단가가 다름.
직원이 줄별로 비교해서 판단.

### 면적당 단가 표시하는 품목
- 포맥스류 (3t포맥스, 5t포맥스 등)
- 현수막류 (각목현수막, 아일렛현수막, 재단현수막 등)
- 철판류 (철판실사, 철판자립 등)
- 후렉스류
- 스티커류 (면적 기준 개당 단가)
- 고무자석 (면적 기준 단가)

### 개당 단가 (면적 표시 안 함)
- pe간판류
- A형간판류
- 3D 제품 (가로×세로×높이 규격)
- 서비스 항목

### 자동 판단 + 수동 예외 방식
시스템이 위 기준으로 자동 분류. 틀린 것만 나중에 대표가 ON/OFF 전환 가능.

## 거래처명 처리

### 저장
정식 법인명 그대로 (이카운트 호환 유지)

### 줄임 이름
별도 매핑 테이블: `주)삼성라코스산업안전` → 화면에 `라코스` 표시
초기 15곳 매핑 (1회성 작업)

### 검색
정식명, 줄임명 둘 다 검색 키로 사용

### 레벨 1 범위
기본은 정식 법인명 그대로 표시. 줄임 매핑은 여력 되면 추가 (필수 아님).

### 위치

## 품명 규칙 유지 방침

현재 이카운트에서 쓰는 품명 규칙 (`[두께][재질]+[부가1]+[부가2]`)을 향후 신규 입력에서도 유지.
강제가 아닌 유도 방식: 조립식 폼 기본 + 직접입력 탈출구.
이유: 이카운트 동기화 호환성 + 과거 데이터와 검색 일관성.
적용 시점: 레벨 2 (견적 작성 화면 만들 때).
