# 견적 입력 UI 전면 개편 설계서

> 작성일: 2026-04-16  
> 상태: 브레인스토밍 완료 → 구현 계획 대기

## 배경

현재 견적 입력 화면의 문제점:
1. **팝업 위에 팝업** — 견적 팝업 → 항목 편집 팝업이 겹겹이 쌓여 답답함
2. **품목명 검색 불일치** — "3t포맥스" vs "포맥스3T" 순서 다르면 검색 안 됨
3. **품명 + 규격 동시 검색 불가** — "포맥스 600*900" 처럼 규격까지 같이 검색해야 의미 있음
4. **입력 공간이 좁음** — 작은 팝업 안에서 입력, 실제 내용은 적은데 공간만 차지
5. **미리보기 가시성 없음** — 텍스트만 나열, 실제 견적서 느낌 없음

## 설계 요약

### 1. 전체 레이아웃

팝업을 거의 전체 화면으로 확장 (95vw × 90vh).

```
┌─────────────────────────────────────────────────────────────┐
│ 견적서 작성 — [거래처명]                            [×] 닫기 │
├────────────────────────────────────┬────────────────────────┤
│                                    │                        │
│  [입력 탭]  [미리보기 탭]           │   과거단가 사이드바     │
│                                    │                        │
│  ▸ 헤더 (거래처, 현장, 날짜)        │   검색: [________] 🔍  │
│                                    │                        │
│  ▸ 품목 테이블 (인라인 입력)        │   거래처 결과           │
│    품명 | 규격 | 수량 | 단가 | 금액  │   ─────────────        │
│    ───────────────────────────     │   포맥스3T 600×900     │
│    (행 추가 버튼)                   │   2025-12  단가 15,000 │
│                                    │   ㎡당 27,778원         │
│  ▸ 합계                            │                        │
│                                    │   다른 거래처 ▸         │
├────────────────────────────────────┴────────────────────────┤
│ [임시저장]                            [저장] [인쇄/PDF]      │
└─────────────────────────────────────────────────────────────┘
```

- 왼쪽 70%: 헤더 + 품목 테이블 (입력 탭) 또는 A4 견적서 (미리보기 탭)
- 오른쪽 30%: 과거단가 사이드바 (입력 탭일 때만 표시)

### 2. 품목 입력 방식

팝업 없이 테이블 행에서 바로 입력하는 인라인 방식.

**입력 흐름:**
1. **품목 선택** — 카테고리 드롭다운 또는 직접 입력
2. **단위 유형 선택** — ㎡(면적) / 개(수량) / m(길이)
   - 단위 유형은 가격 계산 방식만 바꿈
   - 규격 입력 필드는 항상 활성화 (수량 기반이어도 규격이 있을 수 있음)
3. **규격 입력** — 가로(mm) × 세로(mm) 입력 필드 항상 표시
4. **수량 입력** — 숫자 입력
5. **단가** — 자동 계산 또는 수동 입력
6. **금액** — 자동 계산 (수량 × 단가)

**추가 옵션 (아코디언):**
- 행 클릭 시 아래로 펼쳐지는 아코디언에서 부착물/추가옵션 선택
- 별도 팝업 없이 같은 화면에서 처리

**행 관리:**
- "행 추가" 버튼으로 새 품목 추가
- 보통 5~6개, 많을 때도 있음
- 행 삭제, 순서 변경 가능

### 3. 검색 개선 (키워드 분리)

하나의 검색창에서 띄어쓰기로 키워드를 분리하여 검색.

**동작 방식:**
- 입력: `포맥스 3t 600`
- 분리: `[포맥스, 3t, 600]`
- 각 키워드가 `product_name + raw_spec` 합친 텍스트에 모두 포함되면 매칭
- 순서 무관: "3t포맥스" = "포맥스3T" 모두 매칭
- 대소문자 무시

**적용 범위:**
- 과거단가조회 탭 (tab-sales-lookup.html)
- 견적 입력 사이드바 과거단가 검색
- 향후 품목 검색에도 동일 로직 적용

### 4. 기존 데이터 호환성

- UI만 변경, 데이터 구조는 그대로
- 기존에 저장된 견적서도 새 UI에서 정상 표시/편집 가능
- 기존 quotePopup 완전히 교체 (구 UI 코드 제거)
- itemEditPopup 제거 (인라인 입력으로 대체)

### 5. 미리보기 — A4 견적서 양식

미리보기 탭을 실제 견적서 문서 형태로 표시.

**구성:**
- 흰 배경 A4 비율 박스
- 상단: 회사명 + 로고 + "견적서" 제목
- 거래처 정보: 거래처명, 현장명, 날짜
- 품목 테이블: 번호 / 품명 / 규격 / 수량 / 단가 / 금액
- 하단: 합계금액, 부가세, 총액
- 비고란

**동작:**
- 입력 탭에서 항목 추가/수정하면 미리보기에 실시간 반영
- 탭 전환으로 전체 A4를 크게 확인 가능
- 인쇄 / PDF 내보내기 버튼 제공
- 거래처에 실제로 보낼 수 있는 완성도

## 구현 우선순위

1. 전체 레이아웃 변경 (팝업 확장 + 영역 분할)
2. 인라인 품목 테이블 입력
3. 키워드 분리 검색
4. 아코디언 옵션/부착물
5. A4 미리보기
6. 인쇄/PDF 내보내기

## 기술 참고

- 프론트엔드: Alpine.js + Tailwind CSS (기존 스택 유지)
- 검색 API: routes/salesHistory.js 수정 (키워드 분리 로직)
- 인쇄: CSS `@media print` + `window.print()` 또는 서버사이드 PDF 생성
