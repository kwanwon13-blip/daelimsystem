# 매입 OCR — 회사별 완전 분리 + 퀄리티 로드맵 (설계)

작성일: 2026-06-08
관련 목업: `mockups/esm_maeip_ocr.html`, `mockups/company_maeip_ocr.html`

---

## 1. 배경 / 문제

- 현재 **"OCR 자동입력"(statements, 4분할)** 탭이 에스엠·컴퍼니·매입·매출을 한 화면에 섞어 보여줘 실무 사용 불가 — 사용자가 "이상한 구조"로 인식.
- 별도로 **"컴퍼니 매입 OCR"(purchaseOcrApp)** 탭이 따로 있어 OCR 경로가 둘로 혼재.
- 핵심 불만 3가지:
  1. 회사 구분 없이 통째로 보여줘서 정리를 사용자가 떠안음
  2. 회사 자동분류가 신뢰 안 됨
  3. "껍데기(화면)"만 있고 **엔진(인식·매칭) 퀄리티**가 기대에 못 미침

## 2. 목표 (이 프로그램이 잘하는 한 가지)

회사별 매입명세서를 **사진만 넣으면 최소한의 손으로** 정확히 읽어 ERP에 올린다.
- **에스엠 매입 → 이카운트 등록**
- **컴퍼니 매입 → E2E 엑셀** (+ 같은 거래처 묶음)

## 3. 비목표 (이번 범위 아님)

- 매출 / PPTX 시안 — 기존 기능은 **보존**, 건드리지 않음
- 한 사진 안의 여러 명세서 **자동 분리** — 1사진=1명세서, 잘못 들어온 건 **수동 삭제**
- 4분할 통합 화면 — **숨김**(코드·데이터 보존, 되돌리기 가능)

## 4. 핵심 설계 원칙

1. **회사 = 페이지.** 에스엠/컴퍼니는 **완전 별개 페이지**(메뉴·JS·라우트 분리). 한 화면에 같이 안 나옴. AI가 회사를 추측하지 않으므로 **회사 오분류 = 0**.
2. **엔진 우선, 측정 기반.** 화면 뼈대를 먼저 안정화한 뒤, 진짜 자료로 인식 정확도·매칭률을 **숫자로 재며** 개선한다.

## 5. 화면 (A안 — 이카운트식 좌우)

**공통**
- 좌: 그 회사 매입 명세서 목록 (일자 · 거래처 · 합계 · 상태[검토전/확정]). 클릭해 선택.
- 우: 원본 사진 + 품목 표(상품명 · 규격 · 수량 · 단가 · 금액 · 세액 · 상태) + 확정 / 반려 / **🗑 삭제**.
  - 품목 상태: 🟢 매칭(자동) / 🟡 예측(확인 필요) / 🔴 신규(직접 입력 → 다음번 자동 학습)
- 업로드: 드롭 / Ctrl+V, 여러 장 가능(각 1명세서), 진행 표시.
- **1사진 = 1명세서.** 잘못 들어온 건 🗑 삭제.

**에스엠 매입 OCR** (파랑)
- 출력 = **이카운트 등록**, 명세서별 **개별** 등록 (묶기 없음)

**컴퍼니 매입 OCR** (주황)
- 출력 = **E2E 엑셀**, 개별 + **체크박스로 같은 거래처 여러 건 선택 → 한 파일로 묶어서** 받기

## 5.5 매입 자료 입력 경로 (멀티소스) — OCR이 유일한 길이 아님

사용자 매입 자료는 ① 종이·수기 ② 카톡·문자 사진 ③ PDF·엑셀 파일 (전자세금계산서는 **안 받음** → 홈택스 연동 범위 밖).

| 입력 | 처리 방법 | 정확도 | 상태 |
|---|---|---|---|
| 이미지 (종이·수기·카톡사진) | **클로드 비전 OCR** (`claude-sonnet-4-6`) | 들쭉날쭉(측정·개선 대상) | 1단계 구현됨 |
| 엑셀(.xlsx/.csv) | **파일 직접 읽기**(SheetJS), 거래처별 컬럼 매핑 학습 | 높음 | 증분 추가 |
| 텍스트 PDF | **텍스트 직접 추출** | 높음 | 증분 추가 |
| 스캔(이미지) PDF | 페이지를 이미지로 렌더 → OCR | OCR과 동일 | 증분 추가 |

**원칙**: 디지털 원본(엑셀·텍스트PDF)은 OCR 거치지 말고 **그대로 읽어 정확도를 공짜로 확보**. OCR은 이미지 전용 최후수단. 둘 다 같은 라인아이템 구조로 합류 → 같은 검토·매칭·출력 파이프라인.

## 6. 아키텍처 / 컴포넌트

- **신규 페이지**: 에스엠 매입 OCR (탭 HTML + JS + 라우트)
- **개편**: 컴퍼니 매입 OCR → A안 목록형으로 (현재 업로드-단건형 → 좌측 상시 목록형)
- **공유 엔진(재사용)** — 회사를 파라미터로 일반화:
  - OCR 추출 (Claude Vision)
  - 품목 매칭/학습 (`lib/purchase-matcher.js` + `learning-data/`)
  - 거래처 정정 (`lib/vendor-resolver.js`, 사업자번호 기반)
  - 출력 분기: 이카운트 등록 / E2E 엑셀
- **회사별로 다른 것**: 학습 데이터 풀(`01_에스엠매입` vs `03_컴퍼니매입`), 출력 대상, 묶음 기능(컴퍼니만).
- **저장소**: 검증된 기존 매입 파이프라인(company-purchase) 저장구조를 **"회사" 필드로 일반화**해 에스엠·컴퍼니가 같이 쓴다. (1단계 착수 시 `db.js`/company-purchase 저장 코드 확인 후 최종 확정 — db-statements 재사용 대안도 검토)

## 7. 데이터 흐름

```
사진 업로드 → 큐 → OCR 추출(거래처·품목·금액)
  → 거래처 정정(사업자번호) + 품목 매칭(매칭/예측/신규)
  → pending 저장 → 좌측 목록 표시
  → 사용자 검토·수정 (🔴 신규 직접입력 = 학습 반영) → 확정
  → 출력 (에스엠: 이카운트 등록 / 컴퍼니: E2E 엑셀[개별 또는 묶음])
```

## 8. 엔진 퀄리티 — 측정 루프 (2단계 핵심)

- **표본**: 실제 매입 사진 N장(회사·거래처별 대표 케이스) + 사람이 입력한 정답값.
- **측정 지표**:
  - 인식 정확도: 거래처 / 일자 / 품목명 / 규격 / 수량 / 단가 / 금액 **필드별 정답률**
  - 합계 검증: 수량×단가=금액, 라인합=총액 **일치율**
  - 매칭률: 품목 자동매칭(🟢) 비율
  - 등록 성공률: 이카운트 dry-run / E2E 생성 성공률
- **루프**: 측정 → 최다 오류 유형부터 개선(프롬프트 / 이미지 전처리 / 거래처별 템플릿 / 매칭 규칙) → **재측정**. 목표 바(예: 핵심 필드 95%+) 도달까지 반복.

## 9. 에러 / 엣지 케이스

- 한 사진에 다른 명세서 섞임 → 수동 🗑 삭제.
- OCR 실패 → 해당 건 에러 표시, 재시도 / 삭제.
- 품목 매칭 모호 → 🟡 예측으로 표시, 사용자 확정.
- 합계 불일치 → 🔴 경고 표시, 검토 유도.
- ERP 출력은 **dry-run(검증) 먼저 → 확인 후 실제 등록**.

## 10. 영향 / 마이그레이션

- 옛 4분할 statements 탭 → 메뉴에서 **숨김**(코드·데이터 보존, 되돌리기 가능). 매출/시안 등 다른 기능에는 영향 없음.
- 기존 컴퍼니 매입 데이터 → 신규 저장구조로 호환/이관(이관 필요 시 1단계 작업 항목).
- 개발은 **worktree(격리 사본)** 에서 → 라이브 서버 무영향. 반영은 기존 git 배포 흐름(`git-setup.bat` → 서버 pull).

## 11. 로드맵 (확정 순서)

| 단계 | 내용 | 산출물 |
|---|---|---|
| **1. 뼈대** | 깨끗한 두 페이지(에스엠/컴퍼니) 실제 구현 — 업로드→목록→상세→기본 등록까지 동작. 4분할 숨김. | 안정된 무대 |
| **2. 엔진** | 표본 + 측정 루프로 인식 정확도·매칭률 끌어올림. | 측정값 + 목표 바 도달 |
| **3. 마감** | UX 디테일 · 엣지 케이스 · 안정성. | 완성도 |

## 12. 손대는 파일 (1단계 뼈대)

- **신규**: `public/tab-esm-purchase.html`, `public/esm-purchase.js`, `routes/esm-purchase.js`
- **개편**: `public/tab-company-purchase.html`, `public/company-purchase.js` (A안 목록형)
- **수정**: `public/index.html`(탭 등록 / 4분할 숨김), `server.js`(라우트 mount + SSI include)
- **재사용**: `lib/purchase-matcher.js`, `lib/vendor-resolver.js`, 매입 저장/엔진

## 13. 성공 기준

- 에스엠/컴퍼니 각 페이지에서 사진 올림 → 명세서 등록 → 상세 확인 → 출력(이카운트 dry-run / E2E 다운로드) 가능.
- 회사 안 섞임. 1사진=1명세서. 삭제 동작.
- 컴퍼니: 같은 거래처 2건 선택 → 1파일 묶음.
- (2단계) 핵심 필드 인식 정확도를 **측정**할 수 있고 목표 바에 도달.
