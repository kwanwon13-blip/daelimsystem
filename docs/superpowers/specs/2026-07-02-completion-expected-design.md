# 완료예정 표시 (대림컴퍼니 단계) — 설계

## 목적 / 문제
경영관리팀이 **다음날 납품 코스**를 짤 때, 지금은 "이 시안이 진짜 오늘 나오나?"를 **완료 도장이 찍히기 전까지 알 수 없다.** 그래서 예상으로 코스를 잡다가 **안 나오는 것**이 생겨 일정이 어긋난다.
→ 완료 직전, 컴퍼니(공장)가 **"이 시안 오늘 완료예정"** 신호를 미리 켜서, 관리팀이 그걸 보고 코스를 확정하게 한다. (완료일/날짜는 이미 있으나 그건 *이미 끝난 뒤*라 늦음 — 완료예정은 *끝나기 전* 예측 신호)

## 범위 (중요)
- **기존 3단계 그대로**: 디자인팀 → 대림컴퍼니 → 경영관리팀. **새 파이프라인 단계 추가 아님.**
- 대림컴퍼니(factory) 단계 **안에서** 시안(품목)별 `완료예정` ON/OFF 표시만 추가.
- 컴퍼니의 기존 작업 흐름(제작·용접/출력 배정·완료 넘기기)은 **하나도 안 바뀜** — 완료예정은 가볍게 얹히는 버튼 하나.

## 동작
- **토글**: 공장 단계 시안 타일에 `[완료예정]` 버튼. 누르면 ON(초록), 다시 누르면 OFF. **공장(factory) 부서 권한자만** 토글. 보기는 전원.
- **자동 이동 없음**: 완료예정을 켜도 **다음 단계(경영관리)로 자동으로 넘어가지 않는다.** 시안은 대림컴퍼니 칸에 그대로 머문다. 단계 이동은 지금처럼 카드 버튼으로 별도.
- **자연 소멸**: 시안이 실제 완료되어 작업이 경영관리 단계로 넘어가면 완료예정 표시는 의미가 사라진다(사실상 소멸). **매일 리셋은 하지 않는다** — 켜두면 완료될 때까지 유지.
- **부분 기준**: 카드가 "완료예정 섹션"에 뜨는 기준 = **시안이 하나라도 완료예정**(부분 납품 조율 취지). `완료예정 N`(전체 중 몇 개인지) 배지.
- **실시간**: 컴퍼니가 켜/끄면 관리팀 화면에 **즉시 반영**(기존 SSE 알림 재사용). 안 나올 것을 끄면 섹션에서 바로 빠짐.

## 화면 (데스크톱 우선)
- **시안 타일**: `[완료예정]` 버튼(공장권한 x-show) + 완료예정이면 **초록 테두리**.
- **대림컴퍼니 보드 칸 배치**:
  ```
  🔴 긴급              (기존)
  🟢 완료예정 (초록)    ← 새 섹션: 완료예정 시안 있는 카드 모임
  ⚪ 그 외 제작중       (기존)
  ```
  완료예정 카드 = 초록 테두리 + `완료예정 N` 배지. 나머지도 같은 화면에 그대로.
- 관리팀은 같은 칸의 초록 섹션을 보고 내일 코스를 확정.

## 데이터 / API
- 시안(파일) 객체에 `completionExpected`(boolean) 필드 추가. **저장은 기존 `file.team`과 동일 경로**(워크플로 스토어의 파일 객체) — 별도 테이블 불필요, 팀배정과 같은 패턴.
- 토글 라우트: `POST /jobs/:id/files/:fileId/completion-expected { on: true|false }` — **공장 권한 게이트**(팀토글 라우트 `/jobs/:id/files/:fileId/team` 5488과 동일 구조·게이트). 변경 시 이벤트 추가 + notify(실시간).

## 클라이언트 (workflow.js / tab-workflow.html)
- 상태·토글 메서드(`toggleCompletionExpected(file)`), 시안 타일 UI, factory 칸 서브그룹 분류(완료예정/제작중), 초록 테두리 CSS, `완료예정 N` 배지.
- 카드가 완료예정 섹션에 속하는지 = 그 잡의 visualFilesBrief 중 completionExpected 있는지(디자인 단위 접기와 정합 유지, jobTeamCounts 방식 준용).

## 권한
- **완료예정 토글**: 공장(factory) 부서 권한만(`canDeptActOnStage('factory')` 계열).
- **보기**: 전원(관리팀 포함).

## 배포 특성
- 서버(routes/workflow.js) + 클라(workflow.js) 변경 → **서버 재시작 필요** (무재시작 아님). "배포는 일과 후" 원칙 적용.
- workflow.js 정적 캐시버스트 `?v=` 올림.

## 범위 밖 (후속 — YAGNI)
- 모바일 완료예정 토글(데스크톱 확정 후).
- 관리팀 "내일 납품 확정" 체크(양방향 조율) — 필요해지면 그때.
- 완료예정 → 납품/픽업 자동 연동.

## 확정된 결정
- 단위 = 시안(품목)별. / 카드 섹션 기준 = 하나라도 완료예정. / 매일 리셋 없음. / 토글 권한 = 공장만. / 자동 단계이동 없음. / 새 파이프라인 단계 아님(대림컴퍼니 칸 내부 표시).
