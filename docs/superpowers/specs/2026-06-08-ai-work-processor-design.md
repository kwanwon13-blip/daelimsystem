# ERP AI 업무 처리기 (Work Processor) — 설계

작성일: 2026-06-08
상태: 설계 확정 (구현 전 / 검토 대기)

## 1. 한 줄 정의

ERP 내부 AI를 **"업무 처리기"**로 만든다. 잡담 챗봇이 아니라, 직원이 파일을 넣으면 마감·거래명세서·정리 작업을 **안정적으로** 처리해주는 도구다.

- **아는 작업**: 즉시 처리 (지금 퍼시스 마감처럼).
- **처음 하는 작업**: AI가 처리용 "틀(재사용 스크립트)"을 만들어 돌리고, **사장님이 결과를 한 번 확인·승인**하면 그담부턴 "아는 작업"이 된다.

**핵심 원칙 — "그리기"가 아니라 "틀 찍기".** AI가 매번 결과를 새로 그리지 않는다. 한 번 만든 결정적(deterministic) 스크립트가 전월 양식을 복제해 데이터만 채운다 → 매번 똑같이 나온다.

## 2. 해결하려는 문제 (왜 만드나)

기존 AI 챗의 실패 3가지:

1. **지멋대로** — 매번 결과를 새로 생성 → 열·서식·집계가 그때그때 달라짐.
2. **가짜 실행** — 맞는 스킬이 없는데도 엉뚱한 스킬을 돌려 잘못된 파일을 뽑음.
3. **확장 불가** — 거래처마다 개발자가 스킬을 손으로 코딩해야 함.

이 설계가 셋 다 해결한다:

1. 결정적 스크립트 + 템플릿 복제 → 양식이 틀어질 수 없음.
2. 라우터의 1번 규칙 = **"모르면 모른다고 함"** → 가짜 실행 금지.
3. AI가 스크립트를 생성·저장 → 거래처 추가에 개발자 불필요.

## 3. 사용자 / 역할

| 역할 | 하는 일 | 안 해도 되는 일 |
|------|---------|----------------|
| **직원** | 파일 올리고 결과 받음 | 코드·검증·승인 (몰라도 됨) |
| **사장님**(또는 지정 관리자) | 새 작업유형을 승인함에서 한 번 검수 → 등록/고치기/반려 | 코드 읽기 (결과만 봄) |
| **시스템/AI** | 인식 → 생성 → 자동점검 → 실행 → 저장 | — |

**시작 모델: 직원이 시작, 사장님이 검수.**
직원이 파일 + 한 줄 요청 → AI 시도 → (처음 작업이면) 임시 결과를 직원에게 주고 작업유형을 승인함에 올림 → 사장님 검수.

## 4. 구성요소 (각각 하나의 역할만)

### 4.1 입력 — ERP 자체 AI 챗
- 직원이 ERP 자체 AI 챗에 파일 첨부 + `"이거 ○○ 마감해줘"` 같은 한 줄.
- 엔진: **클로드 구독 기반 ERP 자체 챗** (안정화된 print 모드 호출). 외부 클라이언트(Odysseus 등)는 통역서버(claude-bridge) 경유 가능하지만, ERP 자체 챗은 직접 호출한다.

### 4.2 작업 인식 — 라우터
- 첨부 파일 **내용**·요청을 보고 **등록된 작업유형이 있는지** 판단 (기존 내용 스캔 방식 `detectVendorFromAttachments` 활용 — A1만이 아니라 셀 범위 스캔).
- 있으면 → 4.4 저장된 스킬 실행.
- 없으면 → 4.3 처음 작업 경로.
- **1번 규칙: 없으면 가짜로 안 돌린다.** `"처음 보는 작업이에요. 전월 양식 주세요"`라고 정직하게 멈춘다.

### 4.3 처리기 생성 — 처음 작업
- 입력: 원본(이카운트 랜덤파일) + 전월 양식(템플릿) + 한 줄 설명.
- AI가 **변환 스크립트(틀)**를 작성 → 실행 → 결과 파일 생성.
- 스크립트는 **전월 양식을 복제**하고 데이터만 채운다 (퍼시스 방식 그대로). 양식 재해석 ❌.

### 4.4 저장된 스킬 실행 — 아는 작업
- 등록된 스크립트를 그대로 실행 → 결정적 결과.

### 4.5 자동 점검 — 검증 (매번 돌아감)
- 합계 일치(원본 공급가액 합 = 결과 합), 행수 일치(필터 제외), 필수 빈칸 0.
- 통과 → 결과 전달. 실패 → 🚩 `"평소랑 달라요"` → 사장님 확인.
- **아는 작업도 매번 점검** → 거래처가 양식을 바꿔도(드리프트) 조용히 잘못 안 나감.
- 원본 못 읽으면 추정 ❌, "읽기 실패" 보고.

### 4.6 결과 전달
- 아는 작업(승인됨) → 깔끔한 결과.
- 처음 작업(미승인) → `⚠️ 임시 (사장님 확인 전)` 표시 + 작업유형이 승인함으로.

### 4.7 승인 — 기존 결재 시스템에 통합 (별도 탭/화면 없음)
- **방식**: 새 작업유형 승인을 별도 화면으로 만들지 않고, **기존 결재(approval) 시스템에 `AI 작업 승인` 유형으로 통합한다.** 직원이 처음 작업을 돌리면 결과가 **결재 문서 한 건**으로 결재함에 자동 등록된다.
- **위치**: 사장님은 평소 쓰던 **결재함(결재 메뉴)**에서 그대로 처리. 새 탭·새 메뉴 ❌.
- **알림**: 기존 결재 알림 그대로 재사용 — `index.html`의 결재 `approvalNotif` 인앱 팝업 + 사이드바 결재 뱃지가 AI 건도 똑같이 집어준다 (새 알림 코드 ❌).
- **결재 상세에 보여주는 내용**: 사람말 설명 + 결과 미리보기(엑셀) + 자동점검 결과. **코드 ❌.**
- **버튼 (결재 상세화면)**:
  - `✅ 등록` — 승인. 직원이 바로 사용 (일반 결재의 '승인'과 동일)
  - `✏️ 고쳐서 다시` — **일반 결재엔 없는 동작.** "여기 틀렸어, 이렇게" 한 줄 → AI가 스크립트 수정 → 재실행 → 같은 결재 건으로 재검토
  - `❌ 반려`
- 첫 1건은 챗에서도 `"이 방식 저장할까요?"`로 떠서 놓치지 않게.

### 4.8 재사용 스킬 저장소
- 등록된 작업유형 = `SKILL.md` + 스크립트로 저장. 라우터(4.2)가 다음에 찾음.
- 2-PC 동기화: 서버 PC에도 반영돼야 직원들이 씀 (구현 시 경로·동기화 방식 확정).

## 5. 검증 3겹 (직원이 못 미더운 부분을 시스템이 막음)

1. **양식** — 템플릿 복제 → 열·서식 100% 동일 (틀릴 수 없음).
2. **숫자** — 자동점검 (합계·행수·빈칸).
3. **사람** — 사장님이 결과 보고 등록/고치기.

## 6. 범위

- **v1 포함**: 엑셀 기반 마감 / 거래명세서 / 정리. 아는 작업 즉시, 처음 작업 AI생성+사장승인.
- **v1 제외 (나중)**: 잡담 챗봇, PDF 보고서·문서 작성, 외부 시스템 연동.
- **연결**: 기존 퍼시스·나이스텍·하츠 스킬 = 이미 승인된 작업유형. 새 시스템은 이를 **개발자 없이** 확장하는 장치.

## 7. 정직한 한계

- 처음 회사 첫 실행은 AI가 틀을 잘못 만들 수 있음 → 사장님이 한 번 고치면 그담부턴 고정. (옛날은 매번 새로 그려 매번 틀렸고, 새 건 한 번 맞추면 영원히 똑같음.)
- 처음 회사는 **전월 양식**이 있어야 함. 없으면 AI가 추측 안 하고 양식을 요청.
- **원본**은 아무 형태(이카운트 랜덤파일)나 OK. 필요한 건 **출력 양식(템플릿)**.

## 8. 보안 / 안전

- AI 생성 스크립트는 **제한된 실행**: `data/` 파괴 금지, 외부 전송 금지, 출력은 `outputs/` 또는 `data/ai_outputs/`.
- 데이터 파일(`*.db`, `*.json`) 삭제·DROP·TRUNCATE 금지 (CLAUDE.md 규칙).
- 읽기 실패 시 추정 금지, 보고.

## 9. 구현 시 정할 것 (계획 단계로 넘김)

- 스크립트 저장 위치 + 2-PC 동기화 방식.
- 자동점검의 "기대 합계" 산출 방식 (원본 공급가액 열 합 등) — 작업유형별 설정 vs 휴리스틱.
- AI 생성 스크립트 실행 샌드박싱 (보안).
- 라우터의 파일→작업유형 매칭 정밀도 (내용 스캔 기반).
- 기존 결재 모듈 확장: `AI 작업 승인` 유형 추가 + 결과 파일 미리보기 + `고쳐서 다시` 커스텀 동작 수용 범위.

```
직원: [파일 올림] + "이거 ○○ 마감해줘"
         │
         ▼
   ┌───────────────┐   이미 등록된 작업유형인가?
   │   작업 인식    │───────────────┐
   │   (라우터)     │               │
   └───────────────┘               │
      YES │ (아는 작업)          NO │ (처음 하는 작업)
          ▼                         ▼
   ┌────────────┐          ┌──────────────────┐
   │ 저장된 스킬 │          │ AI가 처리기(틀)   │
   │ 즉시 실행   │          │ 만들어 실행       │
   └─────┬──────┘          │ (+전월 템플릿)    │
         │                 └─────────┬────────┘
         └────────────┬──────────────┘
                      ▼
        ┌──────────────────────────────┐
        │  자동 점검 (합계·행수·누락 0)  │  ← 항상
        └──────┬──────────────────┬─────┘
          통과 │             실패 │
               ▼                  ▼
         결과 ✅ 직원에게     "평소랑 달라요" 🚩
                            → 사장님 확인

   ※ 처음 작업이면: 결과는 "⚠️ 임시(미승인)"로 직원에게 +
     작업유형이 [결재함]으로 → 사장님 [등록] →
     다음부터 'YES(아는 작업)' 길로 즉시 처리
```
