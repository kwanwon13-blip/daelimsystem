# 1-2. 단가 이력 추적

- **난이도**: 하 | **효과**: 중
- **참고**: ERPNext Item Price History

## 수정 파일
| 파일 | 작업 |
|------|------|
| `db.js` | `'단가이력': { logs: [] }` 추가 |
| `server.js` | 단가 변경 API에 이력 저장 로직 추가 |
| `public/tab-pricing.html` | 품목별 "단가 이력" 버튼 + 모달 추가 |

## 저장 구조
```js
// 단가 변경 시 자동 기록
{
  id: `ph_${Date.now()}`,
  날짜: new Date().toISOString(),
  업체: vendorId || 'default',
  품목Id: categoryId,
  품목명: categoryName,
  이전단가: { tiers, widthTiers, fixedPrice },
  변경단가: { tiers, widthTiers, fixedPrice },
  변경자: userId,
  사유: reason || ''
}
```

## server.js — 단가 변경 시 이력 저장
```js
// PUT /api/vendor-prices 또는 PUT /api/categories/:id 내부
function savePriceHistory(userId, catId, catName, vendorId, before, after) {
  const hist = db['단가이력'].load();
  hist.logs.push({ id: `ph_${Date.now()}`, 날짜: new Date().toISOString(),
    업체: vendorId || 'default', 품목Id: catId, 품목명: catName,
    이전: before, 변경후: after, 변경자: userId });
  db['단가이력'].save(hist);
}
```

## 프론트 — tab-pricing.html에 이력 버튼
```html
<button @click="openPriceHistory(cat.id)" class="btn-sm">이력</button>
<!-- 모달에서 GET /api/price-history/:categoryId 호출 후 테이블 표시 -->
```
