# 1-5. 자동 백업 시스템

- **난이도**: 하 | **효과**: 상 (데이터 손실 방지)

## 수정 파일
| 파일 | 작업 |
|------|------|
| `자동백업.bat` | 신규 생성 — Windows 작업 스케줄러에 등록 |
| `server.js` | safeJsonSave() 함수 — JSON 저장 시 자동 버전 백업 |

## 자동백업.bat (매일 새벽 3시 실행)
```batch
@echo off
set BACKUP_ROOT=D:\price-list-app\backups
set DATA_DIR=D:\price-list-app\data
set TODAY=%date:~0,4%%date:~5,2%%date:~8,2%

mkdir "%BACKUP_ROOT%\%TODAY%" 2>nul
xcopy "%DATA_DIR%\*.*" "%BACKUP_ROOT%\%TODAY%\" /Y /Q

rem 7일 이전 백업 삭제
forfiles /p "%BACKUP_ROOT%" /d -7 /c "cmd /c if @isdir==TRUE rmdir /s /q @path" 2>nul
echo Backup done: %TODAY%
```

> Windows 작업 스케줄러: 매일 03:00, "자동백업.bat" 실행

## server.js — JSON 저장 시 자동 버전 백업
```js
function safeJsonSave(filePath, data) {
  const backupDir = path.join(__dirname, 'data', '_자동백업');
  if (!fs.existsSync(backupDir)) fs.mkdirSync(backupDir, { recursive: true });

  if (fs.existsSync(filePath)) {
    const name = path.basename(filePath, '.json');
    const ts = new Date().toISOString().replace(/[:.]/g, '-').slice(0, 19);
    fs.copyFileSync(filePath, path.join(backupDir, `${name}_${ts}.json`));

    // 파일별 최근 20개만 유지
    const list = fs.readdirSync(backupDir).filter(f => f.startsWith(name + '_')).sort();
    while (list.length > 20) fs.unlinkSync(path.join(backupDir, list.shift()));
  }

  // 안전한 쓰기 (tmp → rename)
  const tmp = filePath + '.tmp';
  fs.writeFileSync(tmp, JSON.stringify(data, null, 2), 'utf8');
  fs.renameSync(tmp, filePath);
}
```
> db.js의 `.save()` 메서드에서 `fs.writeFileSync` 대신 `safeJsonSave()` 호출로 교체
