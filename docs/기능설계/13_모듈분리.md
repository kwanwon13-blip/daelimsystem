# 3-2. server.js 모듈 분리 (routes/ 폴더)

- **난이도**: 상 | **효과**: 유지보수 향상
- **현황**: server.js ~5250줄 → 분리 후 각 파일 100~400줄

## 목표 구조
```
price-list-app/
├── server.js              ← 메인 (라우터 연결만, ~200줄)
├── middleware/
│   ├── auth.js            ← requireAuth, requireAdmin
│   └── audit.js           ← auditLog() 함수
└── routes/
    ├── auth.js            ← 로그인/회원가입
    ├── categories.js      ← 품목 CRUD
    ├── options.js         ← 옵션 CRUD
    ├── vendors.js         ← 업체 CRUD
    ├── quotes.js          ← 견적 CRUD + PDF
    ├── contacts.js        ← 연락처
    ├── admin.js           ← 사용자/부서 관리
    ├── approvals.js       ← 결재
    ├── attendance.js      ← 근태
    └── dashboard.js       ← 대시보드/통계
```

## server.js 최종 형태
```js
const express = require('express');
const app = express();
app.use(express.json({ limit: '50mb' }));
app.use(express.static('public'));

app.use('/api/auth',       require('./routes/auth'));
app.use('/api/categories', require('./routes/categories'));
app.use('/api/options',    require('./routes/options'));
// ... 등등

app.listen(3000, () => console.log('Server running on :3000'));
```

## 주의사항
- 분리 전 전체 테스트 필수
- db.js require 경로 각 라우터에서 `require('../db')` 로 변경
- session/cookie 미들웨어 server.js 최상단 유지
- 12_모바일반응형.md 작업 시 이것부터 하면 유리
