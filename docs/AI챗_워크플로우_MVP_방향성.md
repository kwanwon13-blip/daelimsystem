# AI챗 / 워크플로우 MVP 방향성

작성일: 2026-06-08

이 문서는 ERP 안에서 개발 중인 `AI 챗`과 `워크플로우`의 방향이 섞이지 않도록 잡아두는 기준선이다.

## 1. 현재 전제

AI 챗과 워크플로우는 둘 다 미완성이다.

다만 둘의 미완성 성격은 다르다.

- AI 챗은 핵심 기능 일부가 이미 동작한다. 이제는 안정화, 기록 유지, 산출물 추적, 템플릿 재사용 UX를 마감해야 한다.
- 워크플로우는 업무 흐름의 방향은 맞지만, 화면과 절차가 아직 직원들이 매일 쓰기에는 복잡하다. 이제는 덜어내고 실제 시안 업무판으로 마감해야 한다.

## 2. 역할 분리

### AI 챗

ERP 안의 내부 AI 작업실이다.

주 역할:

- 엑셀 템플릿 기반 산출물 생성
- PDF/거래명세서/엑셀 분석 및 변환
- 산출 파일 다운로드, 미리보기, 기록 보관
- 반복 작업을 스킬처럼 실행

AI 챗은 워크플로우를 도울 수 있지만, 워크플로우의 기본 업무 흐름이 AI에 의존하면 안 된다.

### 워크플로우

시안/발주/일정 확인을 위한 업무판이다.

주 역할:

- 디자인팀이 시안을 등록한다.
- 관리팀과 공장이 동시에 확인한다.
- 공장은 시안을 보고 가능일, 확인, 조정요청을 남긴다.
- 관리팀은 일정과 납품 흐름을 확인한다.
- 발행된 전달, 다운로드, 취소, 완료 기록이 남는다.

워크플로우의 핵심은 글이 아니라 `시안 이미지`와 `일정 확인`이다.

## 3. AI 챗 마감 기준

이미 준비된 것:

- 엑셀 템플릿 기반 생성 기능
- PDF/거래명세서 추출 및 변환 기능
- 생성 파일 다운로드/미리보기/대화기록 흐름

위 3가지는 새로 시작할 항목이 아니라 `기반 구현 완료`로 본다.
남은 일은 기능을 다시 만드는 것이 아니라, 직원이 실제 업무에서 실패 없이 쓸 수 있도록 안정화하고 연결을 잠그는 것이다.

마감해야 할 것:

- 새로고침해도 대화와 진행 중 작업이 사라지지 않아야 한다.
- 300초를 넘는 작업은 백그라운드 작업으로 계속 진행되고, 완료 후 대화에 다시 붙어야 한다.
- 생성된 파일은 서버 경로와 ERP 다운로드/미리보기로 확실히 연결되어야 한다.
- 템플릿은 매번 업로드하는 것이 아니라 등록, 관리, 재사용이 가능해야 한다.
- 스킬은 버튼식 하드코딩보다 파일과 요청을 보고 적절히 선택되는 구조가 되어야 한다.
- 실패 시 실패 이유와 중간 로그가 남아야 한다.

AI 챗의 MVP 완료 기준은 `직원이 파일을 올리고 요청하면 결과 파일이 ERP 안에 남는 것`이다.

## 4. 워크플로우 MVP 기준

### 기본 화면

기본 화면은 가벼워야 한다.

보여줄 것:

- 대표 시안 이미지
- 회사명
- 프로젝트명
- 희망일/가능일
- 현재 상태
- 긴급 여부

줄일 것:

- 작은 박스 여러 개
- 지나치게 많은 상태 칩
- 항상 보이는 세부 설정
- 담당자 선택
- 제목 직접 입력

### 등록

등록은 팝업에서 단순하게 처리한다.

필수 입력:

- 회사
- 프로젝트
- 희망 완료일
- 시안 파일

기본값:

- 희망 완료일은 워킹데이 +3일
- 등록자는 로그인 사용자
- 제목은 파일명/회사/프로젝트 기반 자동 생성

파일:

- 드래그앤드랍 지원
- 여러 파일 업로드 지원
- 원본 파일명 유지
- 서버의 실제 연도/회사/프로젝트 폴더 규칙을 따른다.

### 상세

상세 화면은 시안을 크게 확인하는 데 집중한다.

필수:

- 큰 시안 미리보기
- 원본 다운로드
- 공장 가능일 입력
- 공장 확인/조정요청
- 관리팀 메모
- 발행/전달 확인

세부 조작은 필요할 때만 펼쳐 보여준다.

### 공장 화면

공장은 복잡한 ERP 기능을 쓰지 않는다.

공장에 필요한 것:

- 시안 크게 보기
- 파일 다운로드
- 가능일 입력
- 확인
- 조정요청

공장은 Cloudflare 터널로 접속할 수 있어야 한다.

### 발행/전달

발행된 전달은 다시 확인할 수 있어야 한다.

보여줄 것:

- 전달 대상
- 파일 수
- 납기
- 다운로드/열람 여부
- 공장 또는 업체 회신 상태
- 발주 취소 여부

잘못 발주한 건은 삭제하지 않고 `발주 취소`로 남긴다.

### 완료

완료된 작업은 사라지지 않는다.

남겨야 할 것:

- 완료일
- 완료 처리자
- 원본 파일 묶음
- 발행/다운로드/확인 기록
- 취소/조정 이력

## 5. 관리자 권한 정의

관리자는 각 단계의 처리를 확인하고, 필요한 단계에서 다음 처리나 취소를 할 수 있어야 한다.

관리자에게 필요한 권한:

- 모든 단계의 확인, 다음 처리, 완료, 보류, 취소 처리
- 잘못된 발주 취소 및 복구
- 전체 작업 조회
- 담당 부서가 아니어도 상태 확인
- 필요한 경우 상태 정정

단, 관리자가 권한을 가진다는 뜻이 공장 확인, 가능일, 조정요청 같은 실제 확인 절차를 없앤다는 뜻은 아니다.

공장 확인이 필요한 흐름은 공장의 가능일/확인/조정요청 기록이 있어야 정상 진행된다.

관리자가 예외 처리한 경우에는 `관리자 처리` 기록이 남아야 한다.

정리하면, 관리자의 권한은 `무기록 우회`가 아니라 `확인/정정/취소/예외 처리 권한`이다.
관리자가 어느 단계에서든 개입할 수는 있어야 하지만, 공장 확인이 필요한 업무를 조용히 건너뛰는 기능으로 만들면 안 된다.

## 6. 지금 넣지 않을 것

MVP에 섞지 않는다.

- 메일 자동발송 완성
- AI가 워크플로우를 자동 진행
- 외부업체별 발주 전체 자동화
- 관리자 무기록 강제 진행
- 모든 구간을 설정창으로 노출
- 제목/담당자/대상자를 매번 직접 선택하게 하는 구조

위 기능들은 2차 이후로 미룬다.

## 7. 추천 마감 순서

1. 워크플로우 기본 화면을 시안 이미지 중심으로 단순화한다.
2. 회사/프로젝트/저장 경로 규칙을 서버 실제 폴더 기준으로 안정화한다.
3. 시안 업로드, 원본 파일명 유지, 큰 이미지 미리보기를 안정화한다.
4. 공장 가능일/확인/조정요청 흐름을 다음 진행 조건과 연결한다.
5. 발행/다운로드/발주 취소/완료 보관을 명확히 한다.
6. AI 챗의 장시간 작업, 템플릿 재사용, 산출물 추적을 안정화한다.
7. 마지막에 AI 챗이 워크플로우 등록이나 요약을 도와주는 연결을 붙인다.

이 순서는 새로 전부 만든다는 뜻이 아니라 `준비된 기능을 실제 업무에서 쓸 수 있는 수준으로 잠그는 순서`다.
즉 1~7번은 "아직 준비 안 된 목록"이 아니라, 이미 들어간 기능을 어떤 순서로 검증하고 마감할지에 대한 체크 순서다.

현재 기준:

- AI 챗의 엑셀 템플릿 생성, PDF/거래명세서 추출, 산출물 다운로드/미리보기/기록 흐름은 기반이 있다.
- AI 챗은 기능 추가보다 새로고침 유지, 장시간 작업, 템플릿 재사용, 실패 로그를 안정화하는 단계다.
- 워크플로우는 방향은 맞고, 화면 단순화와 실제 저장 경로/시안 미리보기/공장 확인 조건을 마감하는 단계다.
- AI 챗이 워크플로우 등록이나 요약을 도와주는 연결은 마지막 단계다.

준비도 기준:

- 1번 기본 화면 단순화: 진행 중. 더 가볍게 만드는 검증이 필요하다.
- 2번 회사/프로젝트/저장 경로: 일부 연결됨. 서버 실제 폴더 규칙 검증이 남았다.
- 3번 시안 업로드/미리보기: 일부 연결됨. 이미지 깨짐, 원본 파일명, 큰 미리보기 검증이 남았다.
- 4번 공장 확인 조건: 방향과 차단 조건은 들어가고 있다. 실제 공장 화면 UX 검증이 남았다.
- 5번 발행/다운로드/취소/완료 보관: 일부 구현됨. 이름과 화면 표현을 더 단순하게 해야 한다.
- 6번 AI 챗 안정화: 기반 있음. 장시간 작업과 템플릿/로그 안정화가 남았다.
- 7번 AI 챗과 워크플로우 연결: 마지막 단계. 앞단이 안정된 뒤 붙인다.

## 8. 판단 기준

새 기능을 넣기 전에 아래 질문을 통과해야 한다.

- 직원이 더 빨리 처리하게 되는가?
- 화면이 더 단순해지는가?
- 시안 이미지와 일정 확인이 더 명확해지는가?
- 기록이 남는가?
- 공장/관리팀/디자인팀의 실제 역할과 맞는가?
- 이 기능이 없어도 MVP가 돌아가는가?

마지막 질문이 `예`라면 MVP 이후로 미룬다.

## 9. 현재 구현 상태 메모

2026-06-08 로컬 기준.

현재 반영 중인 방향:

- 워크플로우 기본 목록은 긴급, 확인, 날짜, 파일 수 중심으로 줄인다.
- 시안별 세부 협의는 기본 노출하지 않고, 선택했을 때만 펼친다.
- 등록 팝업은 회사, 프로젝트, 희망 완료일, 파일 중심으로 둔다.
- 긴급도는 4단계 선택이 아니라 `긴급 요청` 체크로 단순화한다.
- 공장 단계는 시안 수신, 가능일, 일정 상태가 없으면 다음 진행을 막는다.
- 터널로 공장이 열람, 다운로드, 회신한 경우도 시안 수신으로 인정한다.
- 발행/전달 확인 영역은 사라지지 않고, 발행된 건을 다시 확인할 수 있게 둔다.

아직 확인해야 할 것:

- 실제 서버 폴더에서 회사/프로젝트 자동 추천과 저장 경로가 맞는지
- 큰 이미지 업로드 후 썸네일/미리보기가 깨지지 않는지
- 공장 터널 화면에서 다운로드, 가능일 입력, 조정요청이 충분히 단순한지
- 완료 보관함에 원본 파일과 발행 기록이 자연스럽게 남는지
- 관리자 권한으로 확인, 취소, 복구, 정정이 가능한지

서버 반영 전 게이트:

- 로컬 문법 체크 통과
- 워크플로우 등록 팝업에서 시안 등록 가능
- 등록된 시안이 실제 서버 저장 경로에 원본명으로 저장
- 공장 확인 전 단계 진행 차단
- 공장 가능일 저장 후 단계 진행 가능
- 발행된 전달 다시 확인 가능
