# 카톡 이미지 자산 수집 가이드

2만장 이상 이미지를 한 번에 완전 자동으로 처리하려고 하면 카카오톡 화면 변화 때문에 쉽게 멈춘다. 이 작업은 아래처럼 나눠서 진행한다.

```text
카카오톡/톡클라우드에서 묶음 저장
-> C:\CompanyAssets\KakaoInbox
-> Node 감시기가 중복 제거, 썸네일 생성, 인덱스 작성
-> C:\CompanyAssets\KakaoLibrary
```

## 1. 폴더 만들기

```powershell
New-Item -ItemType Directory -Force C:\CompanyAssets\KakaoInbox
New-Item -ItemType Directory -Force C:\CompanyAssets\KakaoLibrary
```

카카오톡 PC 버전이나 톡클라우드에서 사진을 저장할 때 가능하면 `C:\CompanyAssets\KakaoInbox`를 저장 위치로 잡는다.

## 2. 감시기 실행

프로젝트 폴더에서 실행한다.

```powershell
node scripts\kakao-asset-watcher.js --watch
```

기본값:

```text
입력 폴더: C:\CompanyAssets\KakaoInbox
자산 폴더: C:\CompanyAssets\KakaoLibrary
```

다른 폴더를 쓰고 싶으면 이렇게 실행한다.

```powershell
node scripts\kakao-asset-watcher.js --watch --inbox "D:\KakaoInbox" --library "D:\KakaoLibrary"
```

테스트만 하고 파일 복사를 하지 않으려면:

```powershell
node scripts\kakao-asset-watcher.js --dry-run
```

## 3. 생성되는 결과

```text
C:\CompanyAssets\KakaoLibrary
├── asset-index.json       # 전체 자산 목록
├── import-log.csv         # 가져오기/중복/실패 로그
├── originals\YYYY\MM\DD   # 원본 복사본
└── thumbs\YYYY\MM\DD      # 360px 썸네일
```

감시기는 원본 inbox 파일을 삭제하지 않는다. 같은 이미지가 또 들어오면 SHA-256 해시로 중복 판단하고, 새 복사본은 만들지 않는다.

브라우저에서 zip으로 내려받아도 된다. 감시기는 `KakaoInbox` 안의 `.zip` 파일을 열어서 그 안의 이미지까지 자산으로 등록한다.

## 4. Chrome 브라우저 방식

Claude in Chrome이나 일반 Chrome에서 톡클라우드 웹으로 받고 있다면, AutoHotkey보다 브라우저 안에서 버튼 후보를 찾는 편이 낫다.

1. `node scripts\kakao-asset-watcher.js --watch`를 먼저 켜둔다.
2. Chrome에서 톡클라우드 사진 화면을 연다.
3. `F12`를 눌러 DevTools를 연다.
4. Console에 `scripts\kakao-browser-helper-snippet.js` 내용을 붙여넣고 Enter를 누른다.
5. 화면에 `KD0`, `KD1` 같은 주황색 표시가 생기면 다운로드/저장 버튼인지 확인한다.
6. Console에서 수동 클릭한다.

```javascript
kakaoDownloadHelper.click(0)
```

다운로드 후보를 다시 스캔:

```javascript
kakaoDownloadHelper.scan()
```

스크롤하면서 후보 탐색:

```javascript
kakaoDownloadHelper.scrollScan({ steps: 20 })
```

확실히 같은 다운로드 버튼이 반복해서 잡히는 화면에서만 자동 클릭을 쓴다.

```javascript
kakaoDownloadHelper.autoClick({ limit: 20, delayMs: 3000, scrollAfterClick: true })
```

이 스니펫은 외부로 데이터를 보내지 않는다. 현재 로그인된 브라우저 탭 안에서 보이는 버튼 후보를 표시하고 클릭만 한다.

## 5. AutoHotkey 보조 스크립트

`scripts\kakao-download-helper.ahk`는 다운로드 버튼을 대신 눌러주고, `KakaoInbox` 폴더에 새 이미지가 생긴 뒤 파일 크기가 몇 초간 멈출 때까지 기다리는 보조 도구다.

핫키:

```text
F1  도움말
F2  현재 마우스 위치를 다운로드/저장 버튼으로 기억
F3  기억한 위치 클릭 후 다운로드 완료 대기
F4  클릭 없이 다운로드 완료 대기만 수행
F8  KakaoInbox 폴더 열기
F9  일시정지/재개
F12 종료
```

처음에는 50~100장만 선택해서 테스트한다. 감시기 로그에서 `imported`, `duplicate`, `failed` 숫자가 정상인지 본 뒤 500장, 1000장 단위로 키운다.

## 6. 권장 작업 방식

1. `node scripts\kakao-asset-watcher.js --watch`를 켜둔다.
2. PC카톡 또는 톡클라우드에서 한 달치가 아니라 더 작은 묶음부터 저장한다.
3. 감시기 출력에서 `failed=0`인지 확인한다.
4. 문제가 없으면 기간 단위를 키운다.
5. 중간에 멈추면 `import-log.csv`와 `asset-index.json`을 보고 이어서 진행한다.

날짜는 카카오톡 화면에서 읽지 않는다. 가능하면 이미지 EXIF 촬영일을 쓰고, 없으면 파일 수정일 기준으로 `YYYY\MM\DD`에 넣는다.
