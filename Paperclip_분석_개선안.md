# Paperclip 오픈소스 분석 → 우리 프로젝트 개선안

## Paperclip이 뭔가?

AI 에이전트(자동화 봇)들이 회사를 운영하도록 만든 **회사 운영 시스템(Company OS)**입니다.
겉보기엔 업무 관리 툴인데, 내부에 조직도/예산/권한/작업 관리/감사 추적이 다 들어있습니다.

**기술 스택**: Node.js + Express (서버) / React (프론트) / PostgreSQL (DB) / 플러그인 시스템

---

## 우리 프로젝트에 적용 가능한 개선점 (우선순위순)

### ⭐ 1순위 — 바로 적용 가능

#### 1. 감사 로그 (Audit Trail)
- **Paperclip**: 모든 작업(생성/수정/삭제)을 추적. 누가, 언제, 뭘 했는지 기록
- **우리 현황**: 로그 없음. 데이터 수정해도 누가 했는지 모름
- **개선안**: `감사로그.json` 또는 SQLite audit_log 테이블 추가
  ```
  { 시간, 사용자, 행동("품목 수정"), 대상("현수막"), 변경전, 변경후 }
  ```
- **효과**: 문제 발생 시 원인 추적 가능, 실수로 삭제해도 복구 근거 있음

#### 2. 권한 체계 고도화 (Governance)
- **Paperclip**: 역할별 승인 게이트. 중요한 변경은 승인 필요
- **우리 현황**: admin/user 2단계만 있음. 권한이 단순
- **개선안**: 
  - 결재 시스템과 연동 → 견적서 확정, 업체 단가 변경 시 팀장 승인 필요
  - 품목 삭제 같은 위험 동작은 admin만 가능 + 확인 단계 추가
- **효과**: 직원 실수 방지, 중요 데이터 보호

#### 3. 데이터 백업 자동화
- **Paperclip**: config 변경 시 자동 버전 관리, 롤백 가능
- **우리 현황**: 수동 백업만 가능. db.json 날아간 적 있음
- **개선안**: 
  - 매일 자동으로 data/ 폴더 백업 (.bat 스케줄러)
  - JSON 저장 시 이전 버전을 _backup/날짜/ 폴더에 보관
  - 최근 7일치만 유지 (용량 관리)
- **효과**: 데이터 손실 방지 (이미 한번 겪음)

---

### ⭐⭐ 2순위 — 중기 개선

#### 4. 대시보드 강화
- **Paperclip**: 모든 작업/비용/상태를 한눈에 보는 대시보드
- **우리 현황**: 통계 탭이 있지만 기본적
- **개선안**:
  - 홈 화면에 주요 지표 카드: 이번 달 견적 건수, 총 매출액, 최근 활동
  - 최근 변경사항 타임라인 (누가 뭘 했는지)
  - 알림 배지 (미승인 결재, 새 견적 요청 등)
- **효과**: 로그인하자마자 현황 파악 가능

#### 5. 작업 상태 관리 (Ticketing)
- **Paperclip**: 티켓 기반 작업 관리. 상태 추적(대기→진행→완료)
- **우리 현황**: 견적서에 상태(임시/확정)만 있음
- **개선안**:
  - 견적서 워크플로우: 작성 → 검토요청 → 승인 → 발송 → 완료
  - 각 단계에서 담당자 지정 + 알림
  - 견적 목록에 상태 필터/정렬
- **효과**: 견적 진행상황 투명하게 관리

#### 6. 알림 시스템 (이미 계획됨)
- **Paperclip**: 이벤트 기반 알림. 할당/멘션/상태변경 시 자동 알림
- **우리 현황**: 알림 없음 (계획 중)
- **개선안**:
  - 브라우저 알림 (새 결재 요청, 견적 승인 등)
  - 홈 화면 알림 센터
  - 이메일 알림 (SMTP 이미 설정 구조 있음)
- **효과**: 실시간 업무 대응 가능

---

### ⭐⭐⭐ 3순위 — 장기 개선

#### 7. 플러그인/모듈 구조
- **Paperclip**: 플러그인 시스템. 핵심 코드 안 건드리고 기능 추가
- **우리 현황**: server.js 하나에 모든 API가 3,379줄
- **개선안**:
  - API를 모듈별로 분리: routes/categories.js, routes/quotes.js 등
  - 새 기능 추가 시 파일 하나만 만들면 자동 등록
  - server.js는 라우터 연결만 담당
- **효과**: 유지보수 쉬워짐. 기능 추가할 때 충돌 줄어듦

#### 8. API 버전 관리
- **Paperclip**: RESTful API with 체계적 구조 (/api/v1/...)
- **우리 현황**: 버전 없이 /api/categories 직접 사용
- **개선안**: 당장은 불필요. 나중에 모바일 앱이나 외부 연동 시 고려
- **효과**: 호환성 유지하면서 API 업그레이드 가능

#### 9. 멀티테넌시 (다중 회사)
- **Paperclip**: 하나의 서버에서 여러 회사 운영, 데이터 완전 격리
- **우리 현황**: 단일 회사용
- **개선안**: 현재로서는 불필요. 향후 다른 팀/회사에도 시스템 제공 시 고려
- **효과**: 하나의 서버로 여러 현장/법인 관리

## 참고한 Paperclip 핵심 설계 원칙 (우리에게도 유효)

1. **모든 변경은 기록한다** — append-only 로그, 삭제 없음
2. **중요한 건 승인받는다** — 승인 게이트로 실수 방지
3. **예산/한도를 관리한다** — 견적 상한선, 단가 변경 이력
4. **한눈에 파악한다** — 대시보드에 핵심 지표 노출
5. **모듈로 나눈다** — 기능별 분리, 독립적 확장
