# 컨트롤 데몬 (control-daemon) 자동 시작 설정

## 목적
서버 PC(192.168.0.133)에서 **포트 3002**로 항상 돌면서, 다른 PC가 메인 서버(포트 3000)를 원격으로 Start/Stop/Restart 할 수 있게 해주는 데몬.

## 구성 파일
- `control-daemon.js` — 실제 Node.js 데몬
- `control-daemon-watchdog.bat` — 크래시 시 자동 재시작 워치독
- `control-daemon-hidden.vbs` — 워치독을 히든 윈도우로 실행
- `컨트롤데몬시작.bat` — 수동 시작 (테스트용)
- `control-daemon-stop.bat` — 데몬 정지

## 공유 시크릿
현재 값 (watchdog.bat에 하드코딩):
```
e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd
```
로컬 PC 원격 제어 HTML에서 같은 값을 입력해야 인증됨.

## 1회 테스트 실행

서버 PC에서:
```
.\컨트롤데몬시작.bat
```

확인:
```
curl http://localhost:3002/ping
```
응답: `{"ok":true,"service":"control-daemon","version":"1.0"}`

다른 PC(로컬 PC)에서:
```
curl http://192.168.0.133:3002/ping
```
같은 응답이 와야 함. 안 오면 방화벽 또는 바인딩 문제.

## Windows 방화벽 포트 3002 허용

관리자 PowerShell에서:
```powershell
New-NetFirewallRule -DisplayName "Control Daemon (3002)" -Direction Inbound -LocalPort 3002 -Protocol TCP -Action Allow
```

## Windows 작업 스케줄러로 부팅 시 자동 시작

1. `작업 스케줄러` 열기 (taskschd.msc)
2. "작업 만들기" (기본 말고 고급)
3. **일반** 탭:
   - 이름: `Control Daemon`
   - "사용자 로그온 여부에 관계없이 실행" 선택
   - "가장 높은 수준의 권한으로 실행" 체크
4. **트리거** 탭 → 새로 만들기:
   - 작업 시작: `시작할 때` (컴퓨터 부팅 시)
5. **동작** 탭 → 새로 만들기:
   - 동작: `프로그램 시작`
   - 프로그램/스크립트: `wscript.exe`
   - 인수 추가: `"D:\price-list-app\control-daemon-hidden.vbs"` (실제 경로로 교체)
   - 시작 위치: `D:\price-list-app` (실제 경로)
6. **조건** 탭: AC 전원 조건 체크박스 모두 해제
7. **설정** 탭: "작업이 이미 실행되는 경우 다음 규칙 적용" → `새 인스턴스 시작 안 함`
8. 확인 → 관리자 비밀번호 입력

## 재부팅 후 확인
```
curl http://localhost:3002/status?secret=e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd
```

## 로그 확인
`control-daemon.log` 파일이 스크립트 폴더에 생성됨.

## API 요약
모든 POST/GET 호출에 `X-Control-Secret` 헤더 필요 (`/ping` 제외):

| 엔드포인트 | 메소드 | 설명 |
|-----------|--------|------|
| `/ping` | GET | 데몬 생존 확인 (인증 불필요) |
| `/status` | GET | 메인 서버(3000) + 데몬 상태 |
| `/start` | POST | 메인 서버 시작 (VBS 히든 실행) |
| `/stop` | POST | stop.flag + taskkill node |
| `/restart` | POST | 정지 후 5초 뒤 재시작 |

## 예시 curl
```
# 상태
curl -H "X-Control-Secret: e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd" http://192.168.0.133:3002/status

# 시작
curl -X POST -H "X-Control-Secret: e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd" http://192.168.0.133:3002/start

# 정지
curl -X POST -H "X-Control-Secret: e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd" http://192.168.0.133:3002/stop

# 재시작
curl -X POST -H "X-Control-Secret: e0be307ae16da80fa5af012acb3707556ad24c46b604d2e682c3eeace457a0cd" http://192.168.0.133:3002/restart
```
