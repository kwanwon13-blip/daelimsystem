## 무엇을 했나 — 클로드 웹과 똑같은 "안 끊기는 경험"

직원들이 AI 챗을 claude.ai 웹처럼 쓸 수 있도록, **답변 생성을 브라우저 연결과 분리**했습니다.

### 이전 (문제)
- 답변 생성 중 **새로고침/창닫기/다른 PC 접속** → 답변이 끊기고 유실됨
- 생성 중 답변은 완료 시점에만 DB 저장 → 중간에 끊기면 사라짐

### 지금 (해결)
- **서버가 답변을 끝까지 생성**하고 DB에 실시간 저장 (브라우저와 무관)
- F5 새로고침 → 끊긴 자리에서 **이어서 차오름**
- 다른 PC에서 같은 대화 열기 → 진행 중 답변 **그대로 복원**
- 창 닫고 나중에 와도 → **완성된 답변** 표시
- **중단 버튼**만 생성을 멈춤 (단순 연결 끊김은 멈추지 않음)

### 덤으로 개선
- 중단 시 빨간 "오류" 대신 깔끔한 **"⏹ 중단됨"** 표시
- 첨부한 **이미지가 말풍선 안에 썸네일**로 보임 (클로드 웹처럼)
- 모델 기본값 Sonnet 4.6, 잘못된 모델 ID(sonnet-4-7 404) 제거 (이전 작업)
- 채팅 타임아웃 10분, 파일작업 자동 에이전트 라우팅 (이전 작업)

## Phase 2 — 클로드 웹 표면 기능 (완료, 2026-05-31)

| 기능 | 커밋 | 검증 |
|------|------|------|
| thinking(생각 과정) 접이식 박스 | 8b4e1ef | E2E PASS (thinking 텍스트 스트리밍) |
| 메시지 편집(Enter저장/Esc취소) + 재생성 버그 수정 | 30141bd | 단위 검증 |
| HTML/SVG 코드블록 라이브 미리보기 | 3f1e8c9 | 감지 9/9 단위테스트 |
| 프로젝트 지식베이스 | 7c5d2f9 | E2E PASS (지식 주입 확인) |

- **thinking**: claude.ai처럼 "생각 과정"을 접이식 회색 박스로. 생성 중 펼침, 완료 후 접힘, 클릭 토글.
- **편집/재생성**: 편집 Enter=저장·Esc=취소. 재생성 버튼이 깨져있던 버그(msg.id→aiMsg.id) 수정.
- **라이브 미리보기**: AI가 HTML/SVG 코드를 주면 "▶ 미리보기" 버튼 → 우측 샌드박스 패널 즉시 실행.
- **프로젝트 지식**: 프로젝트 목록의 📋 버튼 → 지식 편집 모달. 여기 적은 내용이 그 프로젝트 모든 대화에 자동 주입. (빈 지식 = 기존 동작 변화 0)

통합 스모크 PASS (Phase 1+2 함께 정상).
