# 나이스텍 마감내역서(거래명세서) 생성

나이스텍 거래처의 월별 거래명세서를 만든다. **분석표가 아니라 거래명세서 양식**이다.
전월 마감 파일을 템플릿으로 복제해 데이터만 채우는 방식(회사 양식·합계행·셀 서식 유지). 원본을 못 읽으면 추정하지 말고 읽기 실패를 보고한다.

## 반드시 이 스크립트를 쓸 것

직접 표를 새로 그리거나 정리본/요약 분석표를 만들지 말 것.

```bash
python .claude/skills/nicetech-ledger/scripts/make_nicetech.py --raw <판매현황.xlsx> --template <전월 마감내역서.xlsx> --outdir <출력폴더>
```

- `--raw` (필수): 당월 판매현황 xlsx. 시트 `판매현황`, 1행 기간헤더, 2행 컬럼명, 3행부터 데이터.
- `--template` (선택): 전월 `... 나이스텍 마감내역서 - {거래처}.xlsx`. 거래처별로 매칭, 없으면 최신본 1개를 공용 틀로. **없으면 전월 파일을 요청 — 추정으로 양식을 새로 만들지 않는다.**
- `--outdir` (선택): 생략 시 raw 와 같은 폴더.

## 템플릿(전월 양식) 등록

- 처음 한 번은 **전월 마감내역서 xlsx 를 첨부**해야 양식이 등록된다(`data/ai-skill-templates/nicetech-ledger/`).
- 등록 후에는 **당월 판매현황 파일만** 올려도 같은 양식으로 자동 마감된다.
- 잘못 등록된 양식은 ERP 사이드바 "등록된 마감 양식"에서 삭제할 수 있다.

## 처리 규칙 (퍼시스와 같은 템플릿 복제 방식, 단 구조가 단순)

- **시트 1개(안전)**. 안전/잡자재 구분 없음. 전 품목 포함, `매출할인`만 제외.
- **거래처 분리** = 판매현황 `프로젝트명` 컬럼. 비어 있으면 비고 주소로 판별(비고에 `코닝` → 한국코닝(주) / 그 외 빈칸 → 삼성SDI 기본).
- **E4(거래처명)** = 생성 파일의 거래처명으로 덮어씀. **A4** = 기간.
- 부가세 행 = `부가세별도` 텍스트(수식 아님). 공급가액 G열만 사용.

## 출력

- 거래처별 `... 나이스텍 마감내역서 - {거래처}.xlsx` (안전 1시트). 한글 파일명 NFC 처리.

## 응답

긴 표를 채팅에 붙이지 말고 결과 엑셀만 만든다. 답변은 생성 파일 수·거래처·검산 경고 정도만 짧게.
