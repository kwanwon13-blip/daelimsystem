# 퍼시스 월별 마감 정리

퍼시스 관련 요청은 사용자가 "엑셀로"라고 말하지 않아도 결과물을 엑셀 파일로 만든다. 원본 파일을 못 읽으면 결과를 추정하거나 이전 캐시로 만들지 말고 읽기 실패를 보고한다.

## 반드시 실행할 규칙

월별 마감, 현장별 파일 합산, 거래명세/청구용 정리는 이 스킬의 파이썬 규칙을 우선 사용한다.

```bash
python .claude/skills/persys-ledger/scripts/persys_monthly.py <원본 xlsx 또는 폴더...> --output <결과.xlsx>
```

Agent/CLI 작업 폴더에 첨부 파일이 복사되어 있으면 현재 폴더를 입력으로 써도 된다.

```bash
python <프로젝트루트>/.claude/skills/persys-ledger/scripts/persys_monthly.py . --output 퍼시스_월별정리.xlsx
```

## 파이썬 규칙이 처리하는 것

- 원본 형태 1: 퍼시스 현장별 거래명세 파일의 `안전`, `잡자재` 시트
- 원본 형태 2: 월별 `판매현황` 시트
- 추출 컬럼: 거래일자, 전표번호, 거래처명, 현장명, 구분, 품목코드, 품목, 규격, 수량, 단가, 공급가액, 부가세, 합계, 비고, 원본파일/시트/행
- 부가세가 원본에 값으로 없고 공급가액이 있으면 공급가액의 10%로 계산하되 `부가세기준`에 `10%계산`으로 표시
- 금액 검산은 `수량 * 단가 = 공급가액`, `공급가액 + 부가세 = 합계` 기준으로 확인
- 날짜/품목/금액이 비어 있거나 검산이 맞지 않는 행은 버리지 않고 `확인필요`에 남김

## 출력 시트

- `정리본`: 청구나 공유에 바로 쓸 수 있는 전체 상세 표
- `요약`: 전체/현장별/구분별/품목별 합계
- `확인필요`: 원본에서 불명확하거나 금액 검산이 필요한 행

## 응답

긴 표를 채팅에 붙이지 말고 결과 엑셀 파일을 생성한다. 답변은 파일 생성 완료, 정리 행수, 확인필요 건수 정도만 짧게 말한다. 스킬 폴더명이나 내부 경로는 사용자에게 굳이 노출하지 않는다.
