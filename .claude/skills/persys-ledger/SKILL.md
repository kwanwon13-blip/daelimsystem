# 퍼시스 월별 마감내역서(거래명세서) 생성

퍼시스 관련 요청은 사용자가 "엑셀로"라고 말하지 않아도 결과물을 엑셀 파일로 만든다. 원본을 못 읽으면 추정하거나 이전 캐시로 만들지 말고 읽기 실패를 보고한다.

**방식: 전월 마감내역서(거래명세서) 양식을 그대로 복제한 뒤, 당월 `판매현황` 데이터로 안전/잡자재 2시트를 채운다.** (분석표/요약표를 새로 만드는 게 아니다 — 거래처가 쓰는 마감내역서 양식 그대로 나와야 한다.)

## 반드시 실행할 규칙

```bash
python .claude/skills/persys-ledger/scripts/make_persys.py --raw <판매현황.xlsx> --template <전월 퍼시스-*.xlsx> --outdir <출력폴더>
```

- `--template` 생략 시 raw 파일과 같은 폴더(및 outdir)에서 `퍼시스-*.xlsx` 최신본을 자동 탐색한다.
- `--outdir` 생략 시 raw 파일과 같은 폴더에 출력한다.
- ERP 에이전트는 첨부파일이 작업 폴더에 복사되므로 그 경로를 인자로 넘기면 된다.

## 템플릿(전월 양식) 등록

- 처음 한 번은 **전월 마감내역서 xlsx 를 첨부**해야 양식이 등록된다(`data/ai-skill-templates/persys-ledger/`).
- 등록 후에는 **당월 판매현황 파일만** 올려도 같은 양식으로 자동 마감된다.
- 잘못 등록된 양식은 ERP 사이드바 "등록된 마감 양식"에서 삭제할 수 있다.

## 처리 내용

- 입력: 당월 `판매현황` 시트(거래처명/프로젝트명/품목/수량/단가/공급가액 등)
- 품목 구분: 비고 또는 품목명 규칙으로 `안전`/`잡자재` 2시트에 분배(`매출할인` 제외)
- 부가세가 비어 있고 공급가액이 있으면 10%로 계산
- 출력: **현장(프로젝트)별** `퍼시스-{현장명}-{MM월}.xlsx` (안전/잡자재 2시트)

## 종료 코드(실패 시 ERP가 코드별 안내)

`1`=판매현황 원본 없음 · `2`=전월 템플릿 없음 · `3`='판매현황' 시트 없음 · `4`=템플릿에 '안전/잡자재' 시트 없음 · `5`=데이터 0건 · `6`=결과 파일 0개

## 응답

긴 표를 채팅에 붙이지 말고 결과 엑셀 파일을 생성한다. 답변은 파일 생성 완료, 현장 수, 행수 정도만 짧게 말한다. 스킬 폴더명이나 내부 경로는 굳이 노출하지 않는다.
