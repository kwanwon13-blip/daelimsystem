# 시안 작성 표준 가이드 v1

> 발주 자동화 시스템과 연동되도록 시안 텍스트를 표준 형식으로 작성하는 가이드.
> 컴퍼니 품목마스터(v7)의 등록규칙과 통일됨.

---

## 0. 왜 표준이 필요한가

시안 안에 적은 텍스트가 자동으로 발주 정보로 변환됩니다.
- 디자이너: 시안에 항상 표준 형식으로 텍스트 입력
- 시스템: 텍스트 자동 추출 → 발주서 → e2e 일괄 등록
- 결과: 디자이너 부담 ↓, 발주담당 부담 ↓↓, 정확도 ↑↑

---

## 1. 시안 안에 반드시 들어가야 할 텍스트

### 필수 1줄 — 제품 정보
```
{규격} {상품명} - {수량}개
```

**예시 (좋은 형식)**:
- `600*900 A형철판 - 2개`
- `300*400 포맥스3T - 270개`
- `550파이 덧방원형스티커 - 5개`
- `600*450 고무자석 - 2개`

**나쁜 형식 (자동 추출 실패)**:
- ❌ `600x900 A형 철판 단면 / 두장` (대문자x, 띄어쓰기, 단위가 "장")
- ❌ `사이즈는 600*900이고 두 개` (자연어)

### 필수 1줄 — 납품일
```
납품: {월}/{일}
```
**예시**: `납품: 4/29`

### 다중 디자인일 때 (옵션)
```
각 {개수}개씩 총 {총개수}개
```
**예시**: `각 30개씩 총 270개` (디자인 9종 × 30개)

## 2. 표기 규칙 (v7 신규등록규칙과 동일)

| 항목 | 규칙 | 예시 |
|---|---|---|
| 상품명 | **띄어쓰기 절대 금지** | `3t포맥스` ✅ / `3t 포맥스` ❌ |
| 규격 구분자 | **별표(*) 만 사용** | `600*900` ✅ / `600x900` ❌ |
| 단위 표기 | mm/m 명시할 때만 | `1020mm*50M`, 일반 규격은 단위 생략 |
| 옵션/가공정보 | `+` 기호로 연결 | `3t포맥스+양면`, `포맥스+사방타공` |
| 두께 표기 | 소문자 t 또는 T | `3t포맥스`, `3T포맥스` (둘 다 OK) |
| 두께+규격 동시 | 두께 먼저, 그다음 규격 | `3t포맥스 600*900` ✅ |

## 7. 자동 추출 동작 (참고)

시안저장.jsx 실행 시:

1. 시안 내부 모든 텍스트 프레임 읽기
2. 정규식으로 자동 추출:
   - 규격: `(\d+)\*(\d+)` 또는 `(\d+)파이`
   - 수량: `총\s*(\d+)\s*개`, `(\d+)\s*개`
   - 납품일: `납품\s*[:：]\s*(\d+)\/(\d+)`
   - 두께: `(\d+)t`, `(\d+)T`
3. 분류룰 32개 키워드와 매칭 → 대분류/소분류 자동 결정
4. 마스터에서 OUT-/RAW-/MTL-/SFT-/SUB-/SVC- 코드 매칭
5. 다이얼로그에 자동 채움 → 디자이너 확인 후 [저장] 클릭

## 8. FAQ

**Q. 시안에 텍스트가 그림으로만 있으면?**
- 자동 추출 안 됨. 다이얼로그에서 직접 입력하면 됨.
- 가능하면 텍스트 프레임으로 작업 권장.

**Q. 다중 디자인 시안 (예: 9종 안전픽토)?**
- 한 시안에 9개 디자인 → 텍스트 1개로 표기: `각 30개씩 총 270개`
- 자동 추출이 총수량 270을 잡음

**Q. 신규 상품군이라 사전에 없는 단어인데?**
- 임의로 적어도 됨. 마스터에 신규 후보로 자동 등록됨.
- 운영자가 검토 후 정식 등록.

**Q. 규격이 가로*세로*높이 (3차원)?**
- `600*450*1800` 같이 별표 두 번. 자동 인식됨.

**Q. mm 단위 명시해야 하나?**
- 일반 가로*세로는 mm 생략 (관례). `600*900` 처럼.
- 길이/폭이 다른 단위면 명시: `1020mm*50M`.
